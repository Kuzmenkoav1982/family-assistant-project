# Правила работы с миграциями БД

## Что МОЖНО класть в миграции

✅ **Структурные изменения схемы:**
- `CREATE TABLE`, `ALTER TABLE`, `DROP TABLE`
- `CREATE INDEX`, `CREATE CONSTRAINT`
- Изменение типов колонок, добавление/удаление полей

✅ **Заполнение справочников (seed):**
- Категории расходов
- Список продуктов с КБЖУ
- Базовые тарифные планы
- Любые данные одинаковые для всех пользователей

✅ **Бэкфил данных при миграции схемы:**
- `UPDATE table SET new_column = ... WHERE new_column IS NULL`
- Перенос данных из старой структуры в новую

✅ **Исправления багов в данных, затрагивающих ВСЕХ пользователей:**
- Очистка дубликатов
- Нормализация форматов (email, телефоны)
- Миграция UUID типов

---

## Что НЕЛЬЗЯ класть в миграции

❌ **Ручные операции с данными конкретных пользователей:**
- "Подарить 100₽ Иванову"
- "Активировать подписку для user@mail.ru"
- "Поменять роль конкретного члена семьи"
- "Отправить пуш конкретному пользователю"

❌ **Откаты предыдущих миграций по факту:**
- "Откатить апгрейд Петрова обратно на free"
- "Возврат роли Сидорова с parent на child"

❌ **Тестовые данные для отладки:**
- "Тестовое событие в календаре для конкретной семьи"
- "Демо-профиль здоровья для разработки"

---

## Как делать ручные операции вместо миграций

### Вариант 1 — через админ-панель
Расширить `backend/admin-subscriptions/` или `backend/admin-users/` нужным `action`:
- `gift_subscription` — подарить подписку
- `credit_wallet` — пополнить кошелёк
- `change_role` — изменить роль члена семьи
- `send_notification` — отправить уведомление

Все действия логируются в `admin_actions_log` — для аудита.

### Вариант 2 — разовый SQL-скрипт
Если операция действительно одноразовая, выполнить через `perform_sql_query` напрямую, **не сохраняя в `db_migrations/`**.

---

## Почему это важно

1. **Воспроизводимость** — поднимая dev-копию БД с нуля, миграции применяются последовательно. Ручные операции с продакшен-данными ломают локальную разработку.

2. **Аудит** — действия через админ-функцию логируются с указанием кто, когда и зачем. Миграции этого не дают.

3. **Откат** — если нужно отменить операцию, через админ-функцию это делается легко. Откатить миграцию — почти невозможно.

4. **Размер истории** — на сегодня в проекте 25 ручных миграций из 175. Без правил их будет 100+ за год.
