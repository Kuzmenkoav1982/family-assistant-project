# Goals V1 — Visual QA Pack

Финальный ручной визуальный прогон модуля Целей. Документ закрытый, читается за 10–15 минут, проходится одним человеком без устных пояснений.

Если все секции отмечены `pass` — заполняется `docs/goals/GOALS_V1_SIGNOFF.md`, и модуль уходит во **frozen**.

---

## Подготовка к прогону

1. Открыть проект в hard refresh (Ctrl+F5 / Cmd+Shift+R) — нужно убедиться, что runtime-эффекты сбрасываются.
2. Иметь под рукой:
   - Один аккаунт с **минимум 3 целями разных методик** (SMART / OKR / Wheel).
   - Хотя бы 1 цель с просроченным `deadline`.
   - Хотя бы 1 цель без обновлений >7 дней.
   - Свежий мобильный профиль в DevTools (например, iPhone SE 375×667).
3. Включить **Network = Slow 3G** на 5 секунд, чтобы увидеть skeleton (потом обратно).
4. Включить DevTools → Rendering → **Emulate CSS prefers-reduced-motion: reduce**.
5. Открыть консоль и заранее запустить smoke-tests:
   ```js
   import('@/lib/goals/__smokeTests__').then(m => m.runAllGoalsSmokeTests());
   ```
   Все 7 модулей должны вывести «Итого: N прошло, 0 упало».

## A. Goal detail flows — SMART / OKR / Wheel

Сценарии одинаковые для всех трёх методик. Открыть страницу `/workshop/goal/:id`.

### A1. Normal state
- [ ] Страница открывается без ошибок в консоли
- [ ] Прогресс отображается корректно (число + бар)
- [ ] Цвет витрины соответствует методике:
  - SMART — синий/cyan
  - OKR — фиолетовый/фуксия
  - Wheel — изумрудный/teal
- [ ] Layout не скачет при первой загрузке
- [ ] Mobile (375px) — карточка не ломается, кнопка «Записать» помещается

### A2. Success check-in
- [ ] Поле принимает число и число с запятой (`3,5`)
- [ ] «Записать» (или Enter) сохраняет
- [ ] Кнопка показывает «Сохраняем…» и `disabled` во время запроса
- [ ] Прогресс-бар вспыхивает (pulse + ring + soft glow)
- [ ] Появляется delta-бейдж справа от слова «прогресс» (`+N%` зелёный / `-N%` розовый)
- [ ] Число процентов «докручивается» к новому значению (tick-up)
- [ ] Появляется success-блок «Запись добавлена. … : prev → next»
- [ ] Новая строка появляется в истории и **подсвечивается зелёным** с левой полоской
- [ ] Страница автоматически скроллит к новой строке
- [ ] Подсветка истории гаснет через ~1.8 сек
- [ ] Pulse прогресс-бара гаснет через ~2 сек
- [ ] Success-блок гаснет через ~4 сек

### A3. Validation / Error state
- [ ] Пустое поле → «Укажи новое значение…»
- [ ] «abc» → «Введи число в корректном формате…»
- [ ] То же значение, что и текущее → «Должно отличаться от предыдущего»
- [ ] **Wheel only:** «11» → «Должно быть от 0 до 10»
- [ ] При ошибке фокус возвращается в input
- [ ] При network error (отключить интернет, попробовать «Записать») → «Не удалось сохранить — проверь интернет…»

### A4. Cross-framework consistency
- [ ] При одинаковом действии все три методики ведут себя визуально одинаково (pulse, delta, tick-up, подсветка)
- [ ] Повторный save с тем же значением → нет pulse (validation сработает раньше)
- [ ] Изменить значение на «delta=0 после нормализации» (трудно повторить — заменить число на запись с тем же execution, например 7 → 7 при том же target) → нет pulse

### A5. Runtime hygiene
- [ ] Hard refresh (Ctrl+F5) — никакие pulse/подсветка/delta не висят
- [ ] Подсветка строки одноразовая: после first show — не зажигается без нового save

### A6. Reduced-motion
- [ ] При включённом `prefers-reduced-motion: reduce`:
  - glow/zoom не появляется
  - tick-up мгновенный (без докрутки)
  - delta-бейдж появляется без `zoom-in-95`
  - подсветка строки — есть, но без анимации

## B. Goals Hub — `/workshop`

### B1. Normal state
- [ ] Hero + триада + Hub секция отрисованы без ошибок
- [ ] Summary tiles: «Активных / Средний % / Внимание / Завершено» — числа совпадают с данными
- [ ] Chips-фильтры по умолчанию: активен «Активные»
- [ ] Карточка цели показывает:
  - иконка сферы + framework badge + бейдж сферы
  - title (макс 2 строки)
  - mini progress bar с цветом методики + `%`
  - «3 дн. назад / вчера / сегодня» (last activity)
  - deadline справа внизу (если есть)
- [ ] Клик по карточке → переход в `/workshop/goal/:id`

### B2. Attention state
- [ ] Цель с просроченным дедлайном — карточка с **янтарной рамкой + ring**
- [ ] Чип `просрочено` (розовый) виден
- [ ] Цель без обновлений >7 дней — чип `заброшено` (янтарный)
- [ ] Цель с прогрессом <20% — чип `низкий прогресс` (только маркер, рамка нейтральная)
- [ ] Клик по тайлу «Внимание» → фильтр переключается на `attention`, видны только нужные карточки

### B3. Filter empty
- [ ] Выбрать «OKR» при отсутствии OKR-целей → блок «Нет целей под выбранный фильтр» + кнопка «Сбросить»
- [ ] «Сбросить» → возврат на «Все»

### B4. Empty (нет целей)
- [ ] У нового аккаунта (или временно удалить все цели) → большой dashed-блок «Пока нет целей» + CTA «Создать цель»

### B5. Loading
- [ ] Slow 3G: 4 skeleton-карточки в grid, `animate-pulse`
- [ ] После загрузки skeleton'ы заменяются реальными карточками без layout jump

### B6. Error
- [ ] Заблокировать `?resource=goals` в DevTools → красный alert «Не удалось загрузить цели» + кнопка «Повторить»
- [ ] «Повторить» → загрузка стартует заново

### B7. Mobile (375px)
- [ ] Summary: 2 колонки, не наезжает
- [ ] Chips-фильтры: горизонтальный скролл, ничего не обрезается
- [ ] Карточки: 1 колонка, бейджи помещаются
- [ ] CTA «Цель» и «Дорога» сверху не уплывают за экран

### B8. Sorting
- [ ] «По прогрессу» — сверху самые высокие %
- [ ] «По алфавиту» — кириллица сортируется корректно (А → Я)
- [ ] «По обновлению» — самые свежие сверху
- [ ] «По созданию» — самые новые сверху

## BF. Goals Focus / Execution V1 — верхняя секция Workshop

Появляется **над** Hub и Weekly Review.

### BF1. Очередь есть
- [ ] Заголовок секции: «Что делать сейчас» + подпись «Цели, которым нужно внимание»
- [ ] В правом верхнем углу количество: «N целей в очереди» (склонение)
- [ ] Порядок строк: сначала overdue, потом regressed, потом stale
- [ ] У overdue — розовая рамка/градиент, бейдж «Просрочено», подпись «просрочено на N дней»
- [ ] У regressed — янтарная рамка, бейдж «Откат за неделю», подпись «за неделю -N»
- [ ] У stale — нейтральная рамка, бейдж «Нет активности 7 дней», подпись «без активности N дней»
- [ ] Внутри overdue — самые просроченные сверху
- [ ] Внутри regressed — самое сильное падение сверху
- [ ] Внутри stale — самые заброшенные сверху
- [ ] Клик по строке → `/workshop/goal/:id`
- [ ] CTA-кнопка: «Открыть цель» (для overdue/regressed), «Обновить цель» (для stale)
- [ ] Лимит 6 строк; если в очереди больше — снизу «+ ещё N целей»

### BF2. Очередь пуста (есть цели, но без сигналов)
- [ ] Зелёный dashed-блок «Сегодня всё спокойно — срочных целей нет.»
- [ ] Подсказка «Можно сосредоточиться на росте…»
- [ ] Hub и Weekly Review остаются доступны

### BF3. Нет целей вообще
- [ ] Dashed-блок «Пока нет целей — Focus появится, когда будут активные цели» + CTA «Создать цель»

### BF4. Loading / Error
- [ ] Slow 3G: 3 skeleton-строки. Hub и Weekly Review загружаются независимо
- [ ] Error: красный alert с «Повторить»; Hub живёт даже если Focus упал

### BF5. Дедупликация
- [ ] Цель overdue + stale + regressed одновременно → попадает в Focus **один раз** в группу overdue

### BF6. Фильтр статусов
- [ ] Цель в статусе `done` / `paused` / `archived` НЕ появляется в Focus, даже если у неё просроченный дедлайн

### BF7. Mobile (375px)
- [ ] Бейдж причины переносится на новую строку без поломки
- [ ] CTA «Открыть цель» скрывается, остаётся ChevronRight справа
- [ ] Текст подписи (просрочено на … / без активности …) не наезжает на бейдж

## C. Weekly Review V1.1 — секция в Workshop

### C1. Progress week (есть только рост)
- [ ] Narrative-полоса **изумрудная** (`positive`)
- [ ] Headline: «Хорошая неделя: N целей идут вверх, без откатов.»
- [ ] Nudge: «Продолжить рост: <title>»
- [ ] 4 summary-тайла; активен «Прогресс»
- [ ] Список целей в табе «Прогресс» отсортирован по убыванию delta
- [ ] У каждой строки — зелёный бейдж `+N`
- [ ] «Последние замеры» внизу показывают свежие записи

### C2. Mixed week (прогресс + откат)
- [ ] Narrative-полоса **синяя** (`mixed`)
- [ ] Headline: «Неделя смешанная: N идут вверх, M откатились.»
- [ ] **Первый nudge — на откат** (regressed побеждает прогресс в приоритете)
- [ ] Таб «Откат» содержит ровно `goalsRegressed` записей с розовым `-N`
- [ ] Сортировка регрессий: самая большая просадка сверху

### C3. Attention week (overdue / stale)
- [ ] Narrative-полоса **янтарная** (`attention`)
- [ ] Headline упоминает «требует внимания» / «ждут пересмотра»
- [ ] **Первый nudge — overdue** (если есть просрочка)
- [ ] Таб «Пересмотр»: overdue выше stale выше no-activity
- [ ] У строки есть чип `просрочено` / `заброшено` / `без активности`

### C4. Empty week (есть цели, но нет замеров)
- [ ] Narrative-полоса **slate** (`empty`)
- [ ] Headline: «За последние 7 дней новых замеров не было.»
- [ ] Nudge: «Открыть мои цели»
- [ ] Ниже — большой dashed-блок с CTA «Открыть цели»
- [ ] **summary-тайлы НЕ показываются** в этом состоянии

### C5. No goals at all
- [ ] Narrative секция отсутствует
- [ ] Виден только dashed-блок «Обзор недели появится, когда будут цели»

### C6. Narrative behaviour
- [ ] Тон корректен (`positive` / `mixed` / `attention` / `empty`)
- [ ] Plural работает: «1 цель идёт», «2 цели идут», «5 целей идут», «1 откатилась», «2 откатились»
- [ ] Period-label: «Последние 7 дней · 1 обновление / 2 обновления / 5 обновлений»
- [ ] Длинные title в nudge обрезаются с `…`
- [ ] **Не более 2 nudges**, второй не дублирует первую цель

### C7. Nudge click → нужное место
- [ ] Nudge с `target.kind = 'goal'` → `/workshop/goal/:id`
- [ ] Nudge с `target.kind = 'tab'` → переключает таб (не уводит со страницы)
- [ ] Nudge с `target.kind = 'route'` → переход по path

### C8. Loading / Error
- [ ] Slow 3G — 4 skeleton-тайла + 3 skeleton-строки
- [ ] Блок check-ins не валит весь обзор, если по отдельной цели вернулась ошибка

### C9. Mobile
- [ ] Summary: 2 колонки
- [ ] Narrative-чипы: `flex-wrap`, не обрезаются
- [ ] Строки: текст не наезжает на бейдж справа

## D. Accessibility quick smoke

### D1. Keyboard
- [ ] Tab проходит по логичному порядку: filter chips → sort select → карточки целей
- [ ] Enter в `SmartCheckin` / `OkrCheckin` / `WheelCheckin` — сабмитит
- [ ] В history можно открыть строку клавиатурой

### D2. Live regions
- [ ] При успешном check-in screen reader озвучивает «Запись добавлена»
- [ ] При ошибке озвучивает текст ошибки (role=alert)
- [ ] Изменение прогресса (aria-live на числе %) — озвучивается

### D3. aria-attributes
- [ ] При validation error: `aria-invalid="true"` на input
- [ ] Прогресс-бары: `role="progressbar"` + `aria-valuenow/min/max`
- [ ] Chips-фильтры: `role="tab"` + `aria-selected`

### D4. Reduced motion
- [ ] Tick-up чисел не докручивается
- [ ] Pulse прогресс-бара — без glow/shadow, только тонкий ring
- [ ] Bейдж delta — без `zoom-in-95`

## Naming convention для артефактов

```
goals-<area>-<state>-<device>.png
```

- **area**: `smart` | `okr` | `wheel` | `hub` | `review`
- **state**: `normal` | `success` | `validation` | `attention` | `empty` | `loading` | `error`
- **device**: `desktop` | `mobile` | `tablet` (опционально, по умолчанию desktop)

Примеры:
```
goals-smart-normal-desktop.png
goals-smart-success-desktop.png
goals-smart-validation-mobile.png
goals-hub-attention-desktop.png
goals-review-empty-week-desktop.png
goals-review-narrative-nudge.webm
```

Для видео — расширение `.webm` или `.mp4`. Длительность 10–15 секунд.

Артефакты складывать в одну папку и приложить путь в `GOALS_V1_SIGNOFF.md`.
