# Архив редакции политики конфиденциальности — версия 2026-06-30

Этот пакет фиксирует, какая именно редакция политики была **опубликована**
пользователям и с чем они соглашались.

## Состав пакета
```
docs/legal/privacy-policy/2026-06-30/
  manifest.json                      # метаданные редакции (версия, URL, build, commit)
  privacy-policy-2026-06-30.html     # HTML-слепок с прода (заполняется после выката)
  privacy-policy-2026-06-30.pdf      # PDF той же редакции (заполняется после выката)
  SHA256SUMS.txt                     # контрольные суммы html и pdf
```

## Правило
Слепок снимается **ПОСЛЕ прод-выката**, с боевого URL
`https://nasha-semiya.ru/privacy-policy`, а не из исходного кода.
Иначе это будет артефакт «что собирались опубликовать», а не «что реально
было доступно пользователю».

## Порядок снятия слепка (после выката)
1. Открыть `https://nasha-semiya.ru/privacy-policy`, убедиться, что вверху
   указано «Редакция от 30.06.2026 (версия 2026-06-30)».
2. Сохранить HTML страницы как `privacy-policy-2026-06-30.html`.
3. Распечатать в PDF как `privacy-policy-2026-06-30.pdf`.
4. Посчитать суммы:
   `sha256sum privacy-policy-2026-06-30.html privacy-policy-2026-06-30.pdf > SHA256SUMS.txt`
5. Заполнить в `manifest.json`: `published_at`, `generated_at`, `build_id`;
   при необходимости обновить `commit_hash` на фактический коммит прод-выката.
6. Сменить `status` в `manifest.json` на `published-snapshot`.

## Источник истины по версии
Версия политики задаётся одной env-переменной `CURRENT_PRIVACY_POLICY_VERSION`,
которую читают backend-функции `auth` и `consent`. Фронтенд только отображает
редакцию в тексте политики.
