## 1. Общие сведения

Настоящее руководство описывает порядок установки и развёртывания программного обеспечения «Наша Семья» — веб-платформы для организации семейной жизни. Руководство предназначено для системных администраторов и технических специалистов, осуществляющих развёртывание продукта в производственной среде.

## 3. Архитектура системы

Приложение «Наша Семья» построено на трёхзвенной архитектуре:

```
[Браузер пользователя]
        ↓ HTTPS
[Статический хостинг / CDN]
  (Frontend: React SPA)
        ↓ HTTPS REST API
[Serverless Functions]
  (Backend: Python 3.11)
        ↓ TCP/SSL
[PostgreSQL БД]       [S3 Объектное хранилище]
```

**Компоненты:**
- **Frontend** — одностраничное приложение (SPA) на React 18, собирается Vite и раздаётся как статические файлы HTML/CSS/JS
- **Backend** — набор изолированных функций (Serverless / Cloud Functions) на Python 3.11
- **База данных** — PostgreSQL 15, управляемый сервис
- **Файловое хранилище** — S3-совместимое объектное хранилище для медиафайлов
- **CDN** — раздача статических ресурсов с кешированием

## 4. Установка и развёртывание

### 4.1. Шаг 1. Подготовка исходного кода

1. Получите архив с исходным кодом программного обеспечения (ZIP-архив, поставляемый правообладателем) или клонируйте репозиторий.
2. Распакуйте архив в рабочую директорию:

```bash
unzip nasha-semya.zip -d nasha-semya
cd nasha-semya
```

### 4.2. Шаг 2. Установка зависимостей Frontend

Убедитесь, что на сборочной машине установлен Node.js версии 18+:

```bash
node --version   # должно быть >= 18.0.0
```

Установите зависимости:

```bash
npm install
# или
bun install
```

### 4.3. Шаг 3. Настройка переменных окружения Backend

Для каждой серверной функции необходимо задать переменные окружения в панели управления Yandex Cloud Functions (или в аналогичной Serverless-платформе):

| Переменная | Описание | Пример значения |
|---|---|---|
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql://user:pass@host:5432/dbname` |
| `YANDEX_CLIENT_ID` | Идентификатор OAuth-приложения Яндекс | `xxxxxxxxxxxxxxxx` |
| `YANDEX_CLIENT_SECRET` | Секрет OAuth-приложения Яндекс | `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` |
| `VK_APP_ID` | Идентификатор OAuth-приложения ВКонтакте | `12345678` |
| `VK_APP_SECRET` | Секрет OAuth-приложения ВКонтакте | `xxxxxxxxxxxxxxxxxxxxxxxx` |
| `AWS_ACCESS_KEY_ID` | Ключ доступа к S3-хранилищу | `YCAJExxxxxxxxxxxxxxxxxx` |
| `AWS_SECRET_ACCESS_KEY` | Секретный ключ S3-хранилища | `YCPxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` |
| `NOTIFICATIONS_URL` | URL внутренней функции уведомлений | `https://functions.yandexcloud.net/xxx` |
| `JWT_SECRET` | Секрет для подписи JWT-токенов | `случайная строка >= 32 символов` |

> **Важно:** Никогда не передавайте значения переменных окружения в открытом виде через репозиторий или фронтенд. Все переменные хранятся исключительно в конфигурации серверной стороны.

### 4.4. Шаг 4. Развёртывание базы данных

1. Создайте управляемый кластер PostgreSQL 15 в Yandex Cloud (или на собственном сервере).
2. Создайте базу данных и пользователя с необходимыми правами:

```sql
CREATE DATABASE nasha_semya_db;
CREATE USER nasha_app WITH PASSWORD 'ваш_пароль';
GRANT ALL PRIVILEGES ON DATABASE nasha_semya_db TO nasha_app;
```

3. Примените миграции базы данных. Миграции хранятся в папке `db_migrations/` и выполняются последовательно (V001 → V158):

```bash
# Пример с Flyway:
flyway -url=jdbc:postgresql://host:5432/nasha_semya_db \
       -user=nasha_app \
       -password=ваш_пароль \
       -locations=filesystem:./db_migrations \
       migrate
```

> Всего в поставке содержится **158 миграций**, создающих полную схему базы данных: пользователи, семьи, задачи, календарь, здоровье, финансы, питание, путешествия, дети и прочее.

### 4.5. Шаг 5. Развёртывание Backend-функций

Backend состоит из набора Serverless-функций на Python 3.11. Каждая функция расположена в директории `backend/<имя-функции>/`.

**Для каждой функции:**

1. Загрузите содержимое папки функции в Yandex Cloud Functions (через Yandex CLI или веб-консоль).
2. Укажите точку входа: `index.handler`
3. Установите среду выполнения: **Python 3.11**
4. Задайте переменные окружения (см. п. 4.3)
5. Установите таймаут: **не менее 30 секунд** (для функций AI-анализа — до 60 секунд)
6. Выделите память: **не менее 256 МБ**

**Список функций для развёртывания:**

| Группа | Функции |
|---|---|
| Аутентификация | auth, password-reset, user-management |
| Семья | family-data, family-members, family-invites, family-settings, family-tracker, family-wallet |
| Задачи и события | tasks, calendar-events, events, event-guests, event-expenses, event-tasks, event-wishlist |
| Здоровье | health-profiles, health-records, health-vaccinations, health-medications, health-doctors, health-ai-analysis, health-vitals |
| Питание | nutrition, meal-plans, recipes, generate-diet-plan, diet-progress |
| Финансы | payments, payment-sbp, payments-sber, payments-tbank, family-wallet, purchases |
| Путешествия | trips, trips-ai-recommend |
| Дети | children-data, child-assessment, analyze-development, development-plan |
| AI и аналитика | ai-assistant, generate-image, analytics, page-views |
| Уведомления | notifications-api, push-notifications, telegram-notify |
| Администрирование | admin-users, admin-subscriptions, audit-logger, rate-limiter, data-cleanup |
| Интеграции | alice, domovoy-donations, max-bot, yandex-maps-key |

После успешного развёртывания сформируйте файл `func2url.json` с маппингом имён функций на их URL — он используется фронтендом для вызова API.

### 4.6. Шаг 6. Сборка Frontend

Выполните сборку статического приложения:

```bash
npm run build
# или
bun run build
```

После выполнения в папке `dist/` будут находиться готовые файлы для публикации:
- `index.html` — точка входа
- `assets/` — CSS, JavaScript, шрифты, изображения

### 4.7. Шаг 7. Публикация Frontend

Загрузите содержимое папки `dist/` на статический хостинг или CDN:

- Yandex Object Storage (с настройкой хостинга статического сайта)
- Nginx / Apache (конфигурация для SPA)
- Любой CDN с поддержкой HTTPS

**Пример конфигурации Nginx для SPA:**

```nginx
server {
    listen 443 ssl;
    server_name nasha-semiya.ru;
    root /var/www/nasha-semya/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

## 5. Настройка домена и SSL

1. Привяжите домен (например, `nasha-semiya.ru`) к IP-адресу или CDN вашего хостинга
2. Настройте SSL-сертификат (рекомендуется Let's Encrypt или Yandex Certificate Manager)
3. Настройте редирект с HTTP на HTTPS
4. Убедитесь, что CORS-политики Backend-функций разрешают запросы с вашего домена

## 9. Часто встречающиеся проблемы

| Проблема | Возможная причина | Решение |
|---|---|---|
| Белый экран при открытии | Неверный путь в конфигурации Nginx | Проверить `try_files` для SPA |
| Ошибка 502 при вызове API | Функция не развёрнута или таймаут | Проверить логи функции, увеличить таймаут |
| Ошибка подключения к БД | Неверная строка `DATABASE_URL` | Проверить переменную окружения |
| Не работает OAuth Яндекс | Неверный redirect_uri | Добавить домен в настройки OAuth-приложения |
| Файлы не загружаются | Неверные ключи S3 | Проверить `AWS_ACCESS_KEY_ID` и `AWS_SECRET_ACCESS_KEY` |
