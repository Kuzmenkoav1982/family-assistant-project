# Status Banner — admin runbook

> Короткий пошаговый гайд для админа. Без техники.

---

## Где это

В админке: **`/admin/status-banner`** (нужно сначала залогиниться через `/admin/login`).

## Как создать баннер за 5 шагов

1. Открыть `/admin/status-banner` → **«Создать баннер»** в правом верхнем углу.
2. Выбрать **Тип** (см. таблицу ниже — какой когда использовать).
3. Написать **Заголовок** и **Текст сообщения**.
4. *(необязательно)* указать кнопку «Подробнее» (заполнить **оба** поля: текст и ссылка).
5. Внизу: переключатель **«Включить баннер» → ON** → **«Сохранить»**.

⚠️ Если оставить «Включить баннер» **OFF** — баннер сохранится как черновик и пользователи его не увидят.

---

## Какой тип когда использовать

| Тип | Цвет | Когда использовать | Пример сообщения |
|---|---|---|---|
| **info** | синий | Информация, новости, нейтральные апдейты | «Новый раздел Питание появился в меню» |
| **update** | фиолетовый | Доступна новая версия / новая фича | «Доступна новая версия приложения, обновите страницу» |
| **maintenance** | янтарный | Запланированные технические работы | «С 02:00 до 04:00 МСК будут проводиться работы» |
| **warning** | оранжевый | Деградация части функций, что-то идёт не так | «Загрузка фото временно работает медленнее обычного» |
| **critical** | красный | Полный сбой / срочное действие пользователя | «Сервис временно недоступен, восстановим в течение часа» |

### Дефолты по приоритету (если несколько баннеров активны одновременно)

`critical (100) > warning (50) > maintenance (30) > update (20) > info (10)`

Можно перебить вручную в поле **Приоритет**.

---

## Закрываемый или нет?

| Тип | По умолчанию | Когда менять |
|---|---|---|
| critical | ❌ нельзя закрыть | Оставить как есть, кроме самых редких случаев |
| warning, maintenance, update, info | ✅ можно закрыть | Менять не нужно для большинства случаев |

> **Правило:** если пользователь должен **обязательно** прочитать сообщение прямо сейчас (упало основное действие) — `critical` + non-dismissible. Всё остальное — пусть закрывает.

---

## Расписание (Schedule)

- **Начало** — баннер появится **не раньше** этой даты/времени. Если не задано — появится сразу при включении.
- **Конец** — баннер исчезнет **в этот момент** автоматически. Если не задано — будет висеть пока админ не выключит.

> Часовой пояс — браузера админа. В БД сохраняется как UTC.

**Типичные сценарии:**

- Анонс работ за сутки: `Начало = сегодня вечер` · `Конец = время окончания работ`.
- Постоянное info (например, «приложение в бета-тесте») — оба поля пустые.

## Аудитория

| Audience | Статус в v1 | Кто увидит |
|---|---|---|
| **all** | ✅ работает | Все, включая анонимных посетителей |
| **authenticated** | 🔒 gated (deferred) | Заработает после security mini-sprint |
| **admins** | 🔒 gated (deferred) | Заработает после security mini-sprint |

### Почему gated

Сейчас сервер не умеет проверять «настоящий» auth-токен пользователя — любой клиент мог бы подставить фейковый заголовок и получить чужой контент. До тех пор, пока не появится верифицированная серверная авторизация, мы **физически режем выдачу** баннеров с аудиторией `authenticated` и `admins` через публичное API. В админке такие баннеры можно создавать как заготовки — они сохранятся в БД и автоматически заработают сразу после ближайшего security-спринта, без переделок.

### Что использовать сейчас

Для всех v1-кейсов выбирай **`all`**. Если правда нужно «только своим» — лучше дождаться security mini-sprint, иначе у тебя сохранится черновик, который пользователи не увидят, и можно ввести себя в заблуждение.

## Route scope (на каких страницах)

В поле «Маршруты» — список префиксов URL, по одному в строке (или через запятую).

- **Пустое поле** = глобально, на всех страницах.
- `/portfolio` → на `/portfolio`, `/portfolio/abc`, `/portfolio/abc/sphere/intellect`.
- `/portfolio` НЕ матчит `/portfolio-compare` (граница сегмента).
- Можно несколько: `/portfolio` + `/workshop` — увидят только пользователи на этих разделах.

**Пример:** баннер «Раздел Финансы временно недоступен» с route scope `/finance` — увидят только те, кто открыл этот раздел.

## Предпросмотр

В форме редактирования есть кнопка **«Предпросмотр»**. Она временно покажет баннер **только тебе** (через локальный override в браузере). После сохранения override автоматически снимается.

⚠️ Если закрыть окно браузера с открытым preview — override уйдёт, но если ты вдруг увидел баннер только у себя, а у других нет — нажми **«Снять preview»**.

---

## Включить / выключить уже существующий баннер

Список → нужный баннер → кнопка **«Включить»** или **«Выключить»**. Это переключатель.

При первом включении автоматически проставляется `published_at` (дата первой публикации). При выключении — `unpublished_at` (для аудита).

---

## Удалить баннер

Список → нужный баннер → кнопка с иконкой 🗑️ → подтвердить. Удаление **необратимо** — баннер исчезает из БД полностью, история действий теряется. Если просто не хочешь его показывать — выключи вместо удаления.

---

## Что увидит пользователь

- Узкая полоса под верхним меню. На широком экране — в один ряд, на телефоне — может растянуться на 2-3 строки.
- Слева — иконка типа (направление = смысл, не только цвет).
- Заголовок жирным, текст обычным.
- Если есть кнопка «Подробнее» — кликабельная ссылка.
- Если можно закрыть — крестик справа. Запомнится в браузере пользователя:
  - до конца баннера (если у баннера есть «Конец»),
  - либо на 90 дней (если «Конец» не задан).

---

## Когда баннер исчезает у пользователя

- Админ выключил баннер.
- Прошло время окончания (`endsAt`).
- Пользователь сам закрыл (если dismissible).
- Пользователь зашёл на скрытую страницу (`/login`, `/welcome`, `/presentation` и т.п. — там shell не показывается вообще).

---

## Что делать, если…

| Ситуация | Что делать |
|---|---|
| Баннер не появляется у пользователя | Проверь: 1) `Включён`? 2) попал ли в окно `startsAt`/`endsAt`? 3) audience совпадает? 4) route scope подходит? |
| Хочу обновить текст активного баннера | Открой баннер → измени → Сохранить. Текст обновится у всех пользователей в течение ~60 секунд (поллинг). |
| Хочу временно скрыть, но не удалять | Кнопка **«Выключить»**. |
| Пользователь жалуется, что у него висит старый баннер | Скорее всего, он сам его закрыл и помнит локально. Очистится автоматически по TTL (90 дней или `endsAt`). |
| Сразу несколько баннеров одновременно — какой покажется? | Тот, у которого выше `priority`. При равных — самый свежий по `published_at`. |
