# Редизайн кабинета ребёнка — финальный отчёт

**Проект:** Наша Семья  
**Раздел:** Дети / Кабинет ребёнка  
**Этапов:** 6  
**Статус:** Завершён (условно принят), готов к финальной полировке

---

## Цель редизайна

Превратить кабинет из «реестра занятий с процентами» в **живое детское пространство / паспорт роста ребёнка**, где в одном месте собраны:

- развитие и занятия
- достижения и победы
- цели и мечты
- семейная память
- первые финансовые привычки
- ощущение «это моё личное пространство»

---

## Что было не так в исходном интерфейсе

| Проблема | Последствие |
|---|---|
| Список кружков с процентами | Ощущение учётной таблицы, не жизни ребёнка |
| Нет ответа «что делать дальше?» | Пользователь не понимает следующий шаг |
| Геймификация на первом плане (очки, уровни) | Кабинет похож на игровой профиль |
| Жёсткий золотой блок «мечта» | Ощущение банкового «premium» продукта |
| Достижения как `string[]` без структуры | Нет типов, нет категорий, нет эмоции |
| Блок семьи с emoji-заглушками | Декоративность без смысла |
| 5 экранов с независимой вёрсткой | Нет системы, каждый блок уникален |
| Ложное локальное сохранение историй | Пользователь думает что сохранил, а данных нет |

## Этап 1 — Мастер-экран кабинета ребёнка

**Файл:** `src/components/children/ChildMasterScreen.tsx`

**Цель:** Создать главный экран с правильной иерархией смыслов вместо плоского списка блоков.

**Что было не так:** Исходная `HomeTabContent` — бесконечная лента одинаковых карточек без приоритетов.

**Что изменили:**
- Иерархия: Ребёнок → Мечта → Рост → Шаги → Занятия → Достижения → Семья → Безопасность
- Hero-блок: аватар + имя + возраст. Очки убраны с первого плана — маленький вторичный элемент справа
- Блок «Мечта»: тёплый песочный градиент (`#fff9f0 → #fef3dc`), не «золотой premium»
- Шаги недели: интерактивные чекпойнты отвечают на вопрос «что делать дальше?»
- Блок «Семья»: реальные аватары из `useFamilyMembersContext()`, не emoji-заглушки
- Навигация: 6 компактных табов вместо сетки 3×2 с крупными карточками

**3 сильных решения:**
1. Мечта стоит выше роста — экран начинается с эмоции, не с аналитики
2. Шаги недели закрывают главный UX-пробел: «что делать дальше?»
3. Реальные члены семьи делают кабинет по-настоящему персональным

**Пользовательский эффект:** За 3 секунды видно — чей кабинет, какая мечта, как идёт рост.

## Этап 2 — Экран «Мой рост»

**Файл:** `src/components/children/GrowthScreen.tsx`

**Цель:** Показать развитие ребёнка как понятный, поддерживающий путь, а не как табель с оценками.

**Что было не так:** Простые прогресс-бары по областям без смысла и без следующего шага.

**Что изменили:**
- Адаптивный инсайт вверху: «Отличная динамика — спорт идёт вперёд» vs «Делает первые шаги»
- Подобласти / навыки внутри каждого направления (Чтение, Логика, Внимание)
- Блок «Сильные стороны» — только позитивные формулировки
- Блок «Что можно поддержать» — мягкие советы, не «слабые места»
- Блок «Что влияет на рост» — связь занятий с конкретными результатами
- Блок «Следующий шаг» с разделением: родитель / ребёнок

**3 сильных решения:**
1. Инсайт адаптируется к уровню — не оценивает, а поддерживает
2. «Что поддержать» оформлено как тёплые советы, не как список проблем
3. Занятия связаны с ростом через блок «Что влияет» — закрывает вопрос «зачем?»

**Пользовательский эффект:** Родитель понимает состояние ребёнка без ощущения «его меряют».

## Этап 3 — Экран «Мои достижения»

**Файл:** `src/components/children/AchievementsScreen.tsx`

**Цель:** Создать эмоциональную историю успехов, а не свалку бейджей.

**Что было не так:** `achievements: string[]` в profile_data — плоский массив без структуры, дат, категорий.

**Что изменили:**
- Три типа карточек: **Большое достижение** (диплом/медаль) / **Маленькая победа** (привычка, усилие) / **Памятный момент** (семейный опыт)
- Hero-блок: главное достижение выделено отдельно — экран начинается с гордости
- Блок «Чем я горжусь» — собирает big-достижения в галерею
- Слово семьи (`WarmQuote`) — цитата к каждому достижению
- Категориальные фильтры: Все / Спорт / Учёба / Творчество / Самостоятельность / Семья
- Диалог добавления: тип + категория + значок + название + слово семьи

**3 сильных решения:**
1. Три типа карточек делают экран живым — и официальные победы, и бытовые маленькие успехи
2. Слово семьи в каждом достижении создаёт эмоциональную связь
3. Пустое состояние тёплое и мотивирующее — не сухой «нет данных»

**Пользовательский эффект:** Хочется листать и добавлять — это история, а не архив.

## Этап 4 — Экраны «Мои занятия» и карточка занятия

**Файлы:** `src/components/children/ActivitiesScreen.tsx`, `ActivityDetailScreen.tsx`

**Цель:** Показать занятия как живую часть жизни ребёнка с пользой и результатами, а не как реестр секций.

**Что было не так:** Просто список с названием кружка, расписанием и процентом — без смысла и связи с ростом.

**Что изменили:**

**Список занятий:**
- Инсайт: «Больше всего сейчас — Спорт. Это развивает выносливость и дисциплину» — связь с ростом видна уже в списке
- Карточка занятия: иконка области + польза + расписание + живой след (последний результат + эмодзи настроения)
- Три статуса: Активно / Запланировано / На паузе — с тёплым текстом для каждого
- Блок «Что можно сделать сейчас» — у каких занятий ещё нет результата

**Карточка занятия:**
- Блок «Что даёт ребёнку» — конкретная польза цветом области
- Диалог «Добавить результат»: настроение + что получилось + заметка
- Мягкое состояние паузы: «Всё достигнутое сохранено. Можно вернуться»
- Следующий шаг только для активных занятий

**3 сильных решения:**
1. Польза области читается до открытия карточки — занятия связаны с ростом уже в списке
2. «Живой след» (эмодзи + результат) убирает ощущение административного учёта
3. Статус «На паузе» — позитивный текст, не «брошено»

**Пользовательский эффект:** Открываешь занятия — видишь жизнь ребёнка, а не расписание.

## Этап 5 — Экран «Моя семья / мои корни»

**Файл:** `src/components/children/FamilyRootsScreen.tsx`

**Цель:** Создать тёплое пространство памяти, близости и идентичности ребёнка — не генеалогическое дерево.

**Что было не так:** Блок семьи в мастер-экране — emoji-заглушки без реальных данных.

**Что изменили:**
- Блок «Важные люди»: реальные члены семьи из `useFamilyMembersContext()` с фото/аватарами и тёплыми подписями по роли (Мама → «Всегда поддерживает и любит»)
- Блок «Наши истории»: реальные воспоминания из `useMemoryEntries()` с фото обложками и датами
- Блок «Наши места»: автоматически извлекается `location_label` из воспоминаний + ручное добавление
- Блок «Наши традиции»: реальные традиции из `useFamilyTraditions()`
- Тёплые слова: автоматически появляются если в семье есть Мама или Папа
- Исправлено: ложное локальное сохранение историй заменено честным переходом в `/memory`

**3 сильных решения:**
1. Три реальных API источника — экран живёт на настоящих данных, не на демо
2. Места вытаскиваются из воспоминаний автоматически — данные уже есть, не нужно вводить заново
3. Тёплые подписи по роли создают человечность без лишнего кода

**Пользовательский эффект:** «Вот мои люди, вот мои корни» — а не «вот список родственников».

## Этап 6 — Unified Design Layer

**Файл:** `src/components/children/ui.tsx`

**Цель:** Собрать из пяти хороших, но разрозненных экранов единую, узнаваемую систему.

### UI-аудит: 8 выявленных паттернов-дубликатов

| Паттерн | Кол-во дублей | Расхождения |
|---|---|---|
| Хедер экрана (кнопка назад + заголовок) | 5 | Разные padding, разный фон |
| Белая карточка-секция с заголовком | 14 | Разные rounded, shadow, border |
| Инсайт-блок (однострочная подсказка) | 3 | Разный bg, разный текст |
| Прогресс-бар | 4 | Разная высота (h-1.5/h-2), разные цвета |
| Диалог добавления | 4 | Одинаковая структура, разный overlay/padding |
| Фильтр-чипы | 2 | Одни классы, разные active-состояния |
| Пустое состояние | 7 | Каждый уникален |
| Блок «Следующий шаг» | 3 | Разные градиенты, разная структура |

**Что объединили:** все 8 паттернов → 18 компонентов в `ui.tsx`

**Что сознательно не объединяли:**
- Hero-блок мастер-экрана (специфичен для главного, не повторяется)
- Блок мечты (специфичная бизнес-логика + уникальный визуал)
- Карточки достижений BigAchievementCard/SmallAchievementCard/MomentCard (разная семантика)
- ActivityListCard (специфична для экрана занятий)

### Компоненты ui.tsx

| Компонент | Назначение | Где используется |
|---|---|---|
| `ScreenPage` | Оболочка страницы (серый фон) | Все 6 экранов |

| `AccentCard` | Акцентная карточка с градиентом | Мечта, достижения, результат, следующий шаг |
| `InsightBanner` | Однострочная подсказка | Рост, занятия, мастер-экран |
| `ProgressBar` | Прогресс-бар | Рост, занятия, мастер-экран |
| `MetaRow` | Иконка + метка + значение | Карточка занятия |
| `StepItem` | Один шаг с чекбоксом | Шаги недели |
| `FilterRow` / `FilterChip` | Горизонтальные фильтры | Достижения |

| `DialogSubmit` / `FormField` / `FormInput` / `FormTextarea` / `EmojiPicker` | Элементы форм | Все диалоги |
| `AddRowButton` | Пунктирная кнопка добавления | Достижения, занятия |

### Дизайн-правила системы

**Цвета по ролям:**
```
Фон страницы:     bg-[#f8f9fb]
Карточка:         bg-white + border-slate-100/80
Тёплый акцент:   #fff9f0 → #fef3dc   (мечта, достижения)
Голубой акцент:  #f7fbff → #eef6ff   (моменты, семья, следующий шаг)
Розовый акцент:  #fff1f2 → #fff7ed   (семья hero)
Мятный акцент:   #f0fdf9 → #f0fdfa   (результат занятия)
Зелёный акцент:  #f0fdf4 → #dcfce7   (безопасность)
```

**Типографика:**
```
Заголовок экрана:   text-[18px] font-bold Montserrat
Заголовок секции:   text-[15px] font-bold Montserrat
Заголовок карточки: text-[16–17px] font-bold Montserrat
Body:               text-sm (14px)
Caption / meta:     text-xs (12px) text-slate-400
Micro:              text-[11px] text-slate-400
```

**Отступы:**
```
Страница:           px-4 pt-3 pb-20 space-y-3
Хедер:              px-4 pt-5 pb-4
Карточка внутри:    px-4 pt-4 pb-4
Между элементами:   gap-2.5 / space-y-2 / space-y-2.5
```

**Диалоги:**
```
Mobile:   fixed inset-0 items-end → bottom sheet (rounded-t-3xl)
Desktop:  sm:items-center sm:rounded-3xl → центрированный modal
Клик на оверлей → закрытие
```

**Радиусы:**
```
Карточки:       rounded-2xl
Иконки внутри:  rounded-xl
Мини-элементы:  rounded-lg / rounded-xl
Хедер:          rounded-b-3xl
```

**Типизация:**
```typescript
// Строгий тип имён иконок — только реальные ключи lucide-react
export type IconName = keyof typeof LucideIcons;

// SectionCard.icon?: IconName
// FilterChip.icon?: IconName
```
