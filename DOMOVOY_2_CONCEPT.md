# Домовой 2.0 — концепт AI-проводника по платформе «Наша семья»

**Версия:** 1.0 · 03.06.2026  
**Статус:** Концепт утверждён, MVP в разработке

---

## Одна фраза

> Домовой — это AI-проводник по платформе «Наша семья», который помогает пользователю понять, **где он находится**, **что ему лучше сделать сейчас** и **как пройти нужный путь шаг за шагом**.

---

## Формула

```
сценарный вход  +  контекстный помощник  +  stepper/бусинки  +  карта платформы
```

---

## Проблема

Платформа большая, экосистемная, многослойная.  
Даже внутреннему человеку тяжело быстро понять:

- где он находится
- что вообще доступно
- с чего начать
- что делать дальше
- что он получит от каждого шага

**Текущая модель Домового:** чат — пользователь сам должен догадаться, что спросить.  
**Нужная модель:** Домовой встречает, уточняет цель, ведёт маршрутом, даёт кнопки.

---

## Что меняется

| Сейчас | Домовой 2.0 |
|--------|------------|
| Пустой чат → жди ввода | Встречает + предлагает сценарии |
| Пользователь думает, что спросить | Домовой уточняет цель |
| Только текстовые ответы | Кнопки, deep links, action cards |
| Не знает где пользователь | Context-aware: понимает текущую страницу |
| Нет прогресса | Stepper: показывает шаги и прогресс |
| Нет обзора экосистемы | Карта платформы: всё видно целиком |

---

## Три режима работы

### 1. Быстро сделать
Для тех, кто знает цель.  
_«добавить ребёнка», «открыть расходы», «достижения»_

### 2. Проведи меня
Для тех, кому нужен маршрут.  
_«не знаю с чего начать», «помоги настроить раздел детей»_

### 3. Объясни
Для тех, кому важен смысл.  
_«что я получу», «зачем этот раздел», «чем он полезен семье»_

## 4 интерфейса Домового 2.0

### Экран 1. Входная точка — сценарный старт

**Файл:** `src/components/domovoy/DomovoyEntry.tsx`  
**Назначение:** первый контакт, выбор намерения, не пустой чат.

Что показывает:
- Приветствие + «Чем помочь сегодня?»
- 6–8 крупных сценарных карточек
- Поле свободного запроса (вторично)
- Кнопка «Карта платформы»

Сценарные кнопки:
```
🏠 Настроить семью       👶 Добавить ребёнка
🌱 Развитие ребёнка      💳 Финансы и карта
🔍 Найти раздел          ▶️  Продолжить начатое
🗺️  Карта платформы      ❓  Не знаю, с чего начать
```

---

### Экран 2. Контекстный помощник на странице

**Файл:** `src/components/domovoy/DomovoyContextPanel.tsx`  
**Назначение:** живёт на каждой странице, понимает контекст.

Формат:
- Desktop: боковая панель справа (collapsible)
- Mobile: bottom sheet, вызываемый кнопкой

Блоки:
1. **«Вы сейчас здесь»** — путь / breadcrumb
2. **«Что можно сделать»** — 2–4 action buttons для текущей страницы
3. **«Зачем это»** — короткое объяснение ценности раздела
4. **Прогресс** — если есть незавершённый процесс

Props: `pageKey` — ключ текущей страницы из реестра

---

### Экран 3. Guided flow / stepper

**Файл:** `src/components/domovoy/DomovoyGuide.tsx`  
**Назначение:** сопровождение в длинных сценариях.

Где нужен:
- Onboarding
- Добавление ребёнка
- Настройка детского раздела
- Подключение финансового сценария
- Любые flow из 3+ шагов

Структура stepper:
```
●──○──○──○
|
Шаг 1 из 4: Добавить данные ребёнка
[Краткое объяснение]
[Что вы получите]

[Перейти] [Пропустить] [Объяснить подробнее]
```

Props: `scenarioId` — ключ сценария из реестра

---

### Экран 4. Карта платформы

**Файл:** `src/components/domovoy/DomovoyPlatformMap.tsx`  
**Назначение:** обзор всей экосистемы «Наша семья».

Модули на карте:
```
👨‍👩‍👧 Семья      👶 Дети        💰 Финансы
📷 Память    📚 Библиотека  🏥 Поддержка
⚙️  Настройки  🤖 Домовой
```

Функции:
- Текущее положение подсвечено
- Связи между модулями
- Краткое описание каждого при наведении/тапе
- Быстрый переход в любой модуль
- Рекомендованный маршрут (для новых пользователей)

## 12 ключевых сценариев

| # | Сценарий | Цель пользователя | Действие Домового | Результат |
|---|----------|-------------------|-------------------|-----------|
| 1 | Первый вход | Понять что делать | Встречает, показывает сценарии | Выбирает путь без растерянности |
| 2 | Настроить семью | Создать семейный профиль | Ведёт по шагам, объясняет зачем | Профиль создан без хаоса |
| 3 | Добавить ребёнка | Создать профиль ребёнка | 3–4 шага со stepper | Профиль заполнен, польза понятна |
| 4 | Запустить раздел «Дети» | Активировать детский контур | Показывает структуру, маршрут | Не теряется внутри |
| 5 | Финансовый контроль / детская карта | Управлять детскими финансами | Переводит в модуль, объясняет | Сценарий понятен и управляем |
| 6 | Где я сейчас? | Понять положение в системе | Показывает текущий раздел + путь | Контекст ясен |
| 7 | Что делать дальше? | Не застрять на странице | 2–4 next best actions | Страницы перестают быть тупиками |
| 8 | Многошаговый процесс | Пройти flow без потери | Stepper с прогрессом и CTA | Выше completion rate |
| 9 | Контекстная помощь | Понять что делать именно здесь | Context-aware подсказки | Помощь релевантная, не общая |
| 10 | Возврат к незавершённому | Продолжить начатое | Помнит шаг, предлагает продолжить | Меньше брошенных сценариев |
| 11 | Поиск раздела | Быстро попасть куда нужно | Принимает запрос, переводит | AI = маршрутизатор |
| 12 | Карта платформы | Увидеть экосистему целиком | Открывает карту с текущим положением | Архитектура понятна |

## MVP — что входит и не входит

### Входит в MVP
- Сценарный вход (DomovoyEntry) с 8 сценариями
- Контекстный блок на странице (DomovoyContextPanel) для 10 ключевых разделов
- Guided flow stepper для 4 сценариев: onboarding, ребёнок, дети, финансы
- Карта платформы v1 — 8 модулей
- Deep links / action links по всем модулям
- Базовая память незавершённого процесса (localStorage)
- Реестр страниц и действий

### Не входит в MVP
- Суперсложная персонализация
- Автономное заполнение форм AI-ом
- Агентные цепочки
- Полная аналитическая память поведения
- Голосовой ввод

## Технические требования

Домовому нужны структурные сущности (не только LLM):

| Сущность | Назначение |
|----------|-----------|
| `PAGE_REGISTRY` | Карта разделов с доступными действиями |
| `SCENARIO_REGISTRY` | Реестр сценариев (шаги, deep links) |
| `USER_CONTEXT` | Роль, заполненность профиля, текущая страница |
| `PLATFORM_MAP` | Граф модулей и связей между ними |
| `INCOMPLETE_FLOWS` | localStorage: незавершённые процессы |
| `ACTION_LINKS` | Маршруты для каждого действия |

**Важно:** без структурных данных Домовой будет «красиво отвечать, но плохо вести».

## KPI успеха

### Продуктовые
- Рост активации ключевых модулей после первого входа
- Completion rate многошаговых сценариев (цель: +30%)
- Снижение drop-off в onboarding

### Поведенческие
- Количество сценарных стартов через Домового
- CTR по action buttons (цель: >40%)
- Доля пользователей, дошедших до полезного действия

### Качественные
- «Мне стало понятнее, куда идти»
- Меньше запросов в поддержку по навигации
